# D2C Script Builder — Base

## Trigger prompt (CS fills this in before running)

```
Run script builder.

Brand: [CLIENT NAME]
Total concepts this run: [number — from monthly package or CS decision]
Product / campaign focus: [specific product, SKU, or angle — or "use Priority SKU Queue"]
Persona: [which persona — from PERSONA-STRATEGY.md or STRATEGIC-BRIEF.md — or "use priority persona"]
Awareness stage: [Problem-Aware / Solution-Aware / Product-Aware — or "mixed"]
Primary emotion: [fear of failure / desire for ease / scepticism / urgency / aspiration]
Funnel stage: [TOF cold / MOF warm / BOF retargeting — or "mixed"]
Format: [UGC / Static / Edit-only video / Mixed — or leave blank and Claude will ask]
Existing performance data: [top performer if known — or "use from STRATEGIC-BRIEF.md"]
Extra context: [production constraints, client requests, anything not in files]
```

**Minimum required fields:** Brand + Total concepts.
All other fields can default to the intelligence files if not specified.
If Format is blank, Claude asks before proceeding — never guess.

---

## The 50/50 Rule (hard-coded — never adjust)

Every run splits concepts into two equal buckets:

**Performance bucket (50%)** — Rooted in confirmed account data
- Extends, iterates, or deepens what's already working
- Must cite a specific ad, ROAS figure, or weekly analysis finding
- Goal: become the next evergreen ad

**New ideas bucket (50%)** — Untested angles and territory
- Angles not yet run in the account
- Whitespace from competitor research or persona gaps
- Still requires a hypothesis: "we believe this will work because…"
- Goal: constantly finding new winners

Examples: 10 concepts = 5 performance + 5 new. 6 concepts = 3 + 3. 4 concepts = 2 + 2.

## Step 0 — Read constraints first

Read `[client-folder]/feedback/script-feedback-log.md`.

If this file does not exist (new client, first run):
→ Log: "No feedback log found — applying MUST-READ-SCRIPTING.md constraints only."
→ Proceed using MUST-READ-SCRIPTING.md as the sole constraint source.
→ Do NOT fabricate rejection patterns.

**Universal rejection patterns (always active, regardless of feedback log):**
1. Vague language — if it works for 10 other brands, delete it
2. Non-scroll-stopping hooks — must create tension, curiosity, or specificity in the first second
3. Overly long or written hooks — speak, don't write; no clause before the point
4. Awareness stage mismatch — cold audience getting product-aware copy
5. No specificity — no real product, real situation, or real outcome mentioned
6. Too explanatory — ads are decisions, not tutorials; cut anything that doesn't move persuasion forward
7. Not platform-native — would you stop for this in feed?
8. Weak static execution — statics are 3–6 words; instant clarity or instant curiosity only
9. Swap test fail — a direct competitor could say this unchanged

- `[client-folder]/MUST-READ-SCRIPTING.md` — NON-NEGOTIABLE. Every concept passes this or it doesn't exist.

## Step 2 — Allocate concepts to buckets

**Performance bucket** — draw from:
- "Next 5 Tests" list in weekly analysis (priority)
- Top performing ad in the account (extend or iterate the mechanism)
- Confirmed angle from STRATEGIC-BRIEF.md already showing signal

**New ideas bucket** — draw from:
- Tier 1 or Tier 2 angles from STRATEGIC-BRIEF.md not yet tested in the account
- Whitespace identified in competitor research
- Underserved persona or funnel stage gap

If weekly analysis is missing: flag it. Use STRATEGIC-BRIEF.md only and note the limitation.

## Step 2.5 — Pre-Write Declaration (every concept — no exceptions)

Complete this for every concept before writing a single word of copy.
If any field cannot be filled from real data in the files — STOP and state what is missing.
Do not proceed with a generic version.

```
CONCEPT [N] — PRE-WRITE DECLARATION
─────────────────────────────────────────────
Bucket: [Performance / New Idea]
Persona: [name from STRATEGIC-BRIEF.md + one-line description]
Core tension: [exact phrase from tension column or VOC glossary — not paraphrased]
Brand mechanism that resolves it: [specific differentiator — not "quality" or "great product"]
Hook territory: [fear / curiosity / identity / reward / realisation / seasonal timing]
VOC phrase to use verbatim: [exact quote from VOC glossary in STRATEGIC-BRIEF.md]
Performance basis (if bucket = Performance): [specific ad name + ROAS, or weekly analysis item]
Whitespace basis (if bucket = New Idea): [specific gap from competitor brief or angle tier]
Format: [UGC / Static / Edit-only video]
Why this format: [1 sentence — grounded in test logic or account data]
─────────────────────────────────────────────
```

If any field is vague, invented, or unchecked → do not write it.

## Step 3 — Choose execution type per concept

| Type | When to use |
|---|---|
| **2 statics + 1 edit-only video** | Mixed test — compare static vs video on same concept |
| **3 statics** | Message-led — headline specificity is the creative question |
| **3 edit-only videos** | Existing footage — listicle, mashup, or VO-driven |
| **UGC x3 hooks** | New creator footage needed — talking head, story-driven |

Default priority: static → edit-only → UGC. Test cheaper formats first unless brief specifies otherwise.

### Static variation structure

```
VARIATION [N] — Static — [descriptor]
Headline: [3–6 words — mechanism or use-case — NOT a brand statement or vague adjective]
Subheading: [1–2 compressed outcome lines — what the mechanism means for the customer]
Visual notes: [what must be shown — product in context, key feature, colour coherence]
CTA: [brand CTA or standard]
```

### Edit-only video structure

```
VARIATION [N] — Edit-only video — [descriptor]
Hook — TEXT ON SCREEN / VO: [exact copy]
Visual inspiration: [what footage to use, where to find it]
Body — TEXT ON SCREEN / VO: [copy structured as on-screen beats]
Visual direction: [footage mapping per beat]
Editor notes: [pacing, music direction, format rules]
```

VARIATION 2 — UGC Hook 2 — [REALISATION / IDENTITY / TIMING]
Hook (spoken, 1 sentence):
Opening visual:

VARIATION 3 — UGC Hook 3 — [REWARD / CURIOSITY / SOLUTION-FIRST]
Hook (spoken, 1 sentence):
Opening visual:

---
BODY FLOW (shared across all 3 hooks):
⚠️ THIS IS AN EXAMPLE — CREATOR USES THEIR OWN WORDS

Example script:
[Full body as natural spoken English. First person. One thought per sentence.
 Brand named clearly within first 10 seconds. Mechanism → function → outcome.
 End with brand CTA line.]

Directional beats:
• [Beat 1 — problem context or opening situation]
• [Beat 2 — brand + product introduced]
• [Beat 3 — mechanism named naturally]
• [Beat 4 — function/outcome in the person's language]
• [Beat 5 — CTA]

  Demographics: [from persona in STRATEGIC-BRIEF.md]
  Platform: [Instagram / TikTok]

  • [product clearly visible + well-lit]
  • [specific movement or situation matching the hook tension]
  • [head-to-camera clip for spoken delivery]
  • [close-up of key product feature being described]
  Avoid: [what not to do — bad visuals, mismatched outfit, wrong product, etc.]
```

### Pre-approval check (run before outputting each concept)

```
PRE-APPROVAL CHECK — Concept [N]
[ ] Pre-write declaration completed — all fields from real data, nothing invented?
[ ] Hypothesis names a specific ad, ROAS, or finding — not "research suggests"?
[ ] Hook stops scroll in under 1 second?
[ ] Hook variations attack different psychological entry points (not surface rewrites)?
[ ] Swap test passed — direct competitor cannot say this unchanged?
[ ] At least one VOC phrase used verbatim from STRATEGIC-BRIEF.md glossary?
[ ] Every performance claim confirmed in product specs or confirmed USPs?
[ ] No banned language from MUST-READ-SCRIPTING.md?
[ ] Script reads naturally aloud — no written clauses, no ad language?
```

Do not present any concept that fails a check. Fix it first, then output.

A scannable format the CS can forward for internal review without requiring navigation of the full file.

CONCEPT 1 — [NAME]

Product: [SKU or product name]
Format: [UGC / Static / Edit-only]

CTA: [brand CTA line]

→ Approve / Reject / Amend: _______________

───────────────────────────────────────────────────────────
[repeat for all concepts]
```

Rules:
- Strip all strategy language. Just the product argument.
- "Why this works" = what the CS says out loud when presenting. Not a brief. Like: "This leads with the specific gear failure — specific enough that [audience type] will stop."
- One page total. Compress if it runs longer.

## Hook variation rules (UGC + video)

3 hooks per UGC concept. Each attacks a different psychological entry point — never the same type twice:
- Hook 1: tension / fear / problem — names a specific failure
- Hook 2: realisation / identity / timing — the moment of recognition
- Hook 3: reward / curiosity / solution-first — the "when it works" payoff

Same body flow for all 3. Body does not change between hook variations.

Hook length:
- Static: 3–6 words. Instant clarity OR instant curiosity. No middle ground.
- Video/UGC: 1 punchy spoken sentence. No clause before the point.

---

## Process rules (non-negotiable)

1. Never write a concept before completing the Pre-Write Declaration
2. Never fabricate account insights — if data is missing, say so and stop
3. Never output work that fails any Pre-Approval Check item — fix it first
4. Never push to Notion automatically — always hold for CS sign-off
5. Client-Ready Summary is always produced — never skip it
6. If STRATEGIC-BRIEF.md is missing — stop and tell the CS to run strategic-brief-builder first
